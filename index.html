Guía de Laboratorio N°11 • Modelado de BD

¿Qué es una Vista?

Son como "fotos" de la información en un instante determinado. Funcionan como tablas virtuales basadas en el resultado de una consulta SQL, permitiendo acceder a datos complejos de forma rápida y segura sin manipular las tablas originales.

Sintaxis Básica

CREATE VIEW vista_nombre AS
SELECT columnas
FROM tablas
WHERE condiciones;
01 REQUERIMIENTO

Analizar qué datos necesita el usuario final.

02 QUERY SQL

Crear el SELECT uniendo (JOIN) las tablas necesarias.

03 CREATE VIEW

Encapsular la consulta en un objeto de vista.

04 ABSTRACCIÓN

El usuario consulta la vista como si fuera una tabla.

CASO: CLÍNICA TODOSALUD

V_PACIENTEENFERMEDAD

Unión de Tablas: Paciente + Diagnóstico + Enfermedad

CREATE VIEW v_pacienteenfermedad AS
SELECT p.nombresP, e.nombreE, d.fechaD
FROM paciente p
JOIN diagnostico d ON p.idPac = d.idPac
JOIN enfermedad e ON d.idEnf = e.idEnf;

Permite ver el historial médico sin lidiar con IDs complejos.

Beneficios "Chad"

  • Seguridad Oculta columnas sensibles de las tablas base.
  • Simplicidad Consultas de 100 líneas se resumen en un "SELECT * FROM vista".
  • Consistencia Si la lógica cambia, solo editas la vista, no la app.
